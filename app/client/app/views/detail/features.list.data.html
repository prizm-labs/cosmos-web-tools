AngularJS & Backbone

AngularJS + Backbone.js =

AngularJS for views, Backbone for models
The awesome thing about the Parse Javascript SDK is that it's a fork of Backbone, so we can make use of the kickass Model and Collection architecture, but still use AngularJS for all UI events and views (this is where Backbone loses out to other frameworks).

The best way to stay organised is to separate each Parse Object definition into a separate file in aÂ data folder. Each definition is a factory of an overall Angular module we have called 'ExternalDataServices' which you'll see is first declared above in #2. Extending the 'ExternalDataServices' module with new factories is easy, just leave out the square brackets. Here's what our folder structure looks like with this architecture.

|-- app
  |-- scripts
    |-- modules
      |-- data
        // this is where all your model definitions 
        |-- Monster.js
And this is what a Parse Object definition would look like:

Monsters.js 
// Monster.js

// reference the module we declared earlier
angular.module('ExternalDataServices')

// add a factory
.factory('MonsterService', ['ParseQueryAngular', function(ParseQueryAngular) {

  // use the extendAngular instead of just extend to gain enhanced Angular functionality

  var Monster = Parse.Object.extendAngular({
    className:"Monster",
    setScaryMove: function(move) {
      this.set('scaryMove',move);
      return this;
    }
  });
  

  // use the extendAngular instead of just extend to gain enhanced Angular functionality

  var Monsters = Parse.Collection.extendAngular({
    model: Monster,
    loadMonstersWithMove: function(move) {
      this.query = (new Parse.Query(Monster));

      // use the enhanced load() function to fetch the collection
      return this.load();
    }
  });

  // Return a simple API : model or collection.
  return {
    model: Monster,
    collection: Monsters
  }

}])

Now wherever we need a Monster model or collection, like in the directive below, we can instantiate it using the MonsterService like this:

// SomeDirective.js

// inject the MonsterService into your directive

angular.module('foo',[]).directive('bar',['MonsterService',function(

  return {
    controller: ['$scope','$element','$attrs', function($scope, $element, $attrs) {

      // new up a collection so we can use it to perform a query
      var monsters = new MonsterService.collection;

      // perform the Parse.Query
      $scope.monsters = monsters.loadMonstersWithMove('fireBreath');

      // and remember its a promise, so we can add synchronous logic here with a .then()

    }
  }]

}])

A great side-effect of injecting your Parse data services only when you need it is that AngularJS's awesome dependency injection system knows when to remove this stuff from memory. Also, if you build your app in a modular way, in Angular 2.0 you will also be able to take advantage of lazy loading modules which will speed up the load-time of your app tremendously.
